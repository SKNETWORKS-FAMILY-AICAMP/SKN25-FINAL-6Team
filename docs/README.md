# Docs

이 디렉터리는 프로젝트의 세부 문서를 관리하는 공간입니다.

현재는 아직 구현 코드와 SQL이 없는 초기 단계이므로, 문서 역시 "최종 명세"가 아니라 설계 방향과 작성 기준을 먼저 고정하는 용도로 채웁니다.

## 문서 구성

- `architecture/`: 시스템 구조와 책임 분리
- `requirements/`: 제품/운영 요구사항
- `erd/`: 데이터 모델과 테이블 관계
- `api/`: 외부/내부 API 명세
- `deployment/`: 외부 PostgreSQL, 이후 Docker/Supabase 배포 절차

## 현재 문서 원칙

- 구현되지 않은 내용은 확정 사실처럼 쓰지 않습니다.
- 현재 기준은 외부 PostgreSQL + `psql` 환경입니다.
- 코드 예시보다 책임, 입력/출력, 연결 관계를 우선 정리합니다.
- 추후 실제 구현이 나오면 README 요약과 `docs/` 상세 문서를 함께 맞춥니다.
