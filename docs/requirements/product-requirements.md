# Product Requirements

## 목표

게임 고객 문의 처리 시간을 줄이고, 반복 문의는 자동화하며, 운영자가 판단해야 하는 문의는 더 빨리 식별하는 것이 목표입니다.

## 주요 사용자

- 게임 이용자
- CS 운영자
- 운영 분석 담당자

## 핵심 요구사항

- 반복 문의를 카테고리별로 분류할 수 있어야 함
- 문의에 대한 초안 응답을 생성할 수 있어야 함
- 운영 검토가 필요한 문의를 자동으로 걸러낼 수 있어야 함
- 운영자가 승인/수정/반려를 수행할 수 있어야 함
- 누적된 문의와 피드백을 바탕으로 운영 리포트를 생성할 수 있어야 함

## 현재 범위 밖

- 고객 대상 완전 자동 환불 처리
- 운영 공지 자동 게시
- DB 스키마 자동 변경
