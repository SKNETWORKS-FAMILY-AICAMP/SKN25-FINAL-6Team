# Non Functional Requirements

## 기본 원칙

- DB 연결은 환경 변수 기반이어야 함
- 외부 PostgreSQL 기준으로 로컬 검증이 가능해야 함
- 구조상 챗봇, 배치, 인사이트 모듈이 분리되어야 함
- 로그와 운영 피드백이 남아야 함

## 품질 기준 초안

- 자동 응답 여부는 신뢰도와 안전성 기준을 통과해야 함
- 오류가 발생해도 원인 추적이 가능해야 함
- 배치 작업은 재실행 가능해야 함
- 추후 Supabase 기반 배포 구조로 이관 가능해야 함

## 보안 기준 초안

- 비밀키는 `.env` 또는 배포 시크릿으로만 관리
- 운영용 DB 계정 권한은 최소화
- 개인정보성 필드는 로그와 응답에서 별도 검토 필요
